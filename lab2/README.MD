## Архитектура модели (baseline — AvgPool)

```
Input (N, 32, 32, 3)  uint8
       ↓
   Normalize            /255 → channel mean/std → permute NHWC→NCHW
       ↓
(N, 3, 32, 32)
       ↓
  Conv2d(3→32, 5×5, stride=4, pad=2)   → (N, 32, 8, 8)
  ReLU
       ↓
  Conv2d(32→64, 3×3, stride=1, pad=1)  → (N, 64, 8, 8)
  ReLU
       ↓
  AvgPool2d(4)                          → (N, 64, 2, 2)
  Flatten                               → (N, 256)
       ↓
  Linear(256 → 3)
       ↓
(N, 3)  logits
```

| Слой | Выход | Параметры |
|------|-------|-----------|
| Normalize | (3, 32, 32) | 0 |
| Conv2d(3→32, 5×5, s=4) | (32, 8, 8) | 2 432 |
| Conv2d(32→64, 3×3) | (64, 8, 8) | 18 496 |
| AvgPool2d(4) | (64, 2, 2) | 0 |
| Linear(256→3) | (3,) | 771 |
| **Итого** | | **21 699** |

## Сравнение стратегий пуллинга

| Конфигурация | Pooling | lr | batch | epochs | Train Acc | Test Acc | Комментарий |
|---|---|---|---|---|---|---|---|
| Conv(32)+Conv(64), AvgPool | AvgPool2d(4) | 5e-3 | 128 | 500 | 100.00% | 89.33% | Baseline |
| Conv(32)+Conv(64), MaxPool | MaxPool2d(4) | 5e-3 | 128 | 500 | — | — | |
| Conv(32)+Conv(64), stride  | stride=4 (нет отд. пуллинга) | 5e-3 | 128 | 500 | — | — | |

> Строки MaxPool и stride заполняются после запуска экспериментов.

**Ожидаемые отличия:**
- `MaxPool` — сохраняет самый сильный сигнал, чувствителен к выбросам.
- `AvgPool` — сглаживает признаки, часто чуть менее точен, но более устойчив.
- `stride` — уменьшает размерность уже в свёртке, меньше параметров на пуллинг-слой.

## Loss Landscape

Визуализация поверхности функции потерь вдоль двух случайных направлений в пространстве весов:

- **1D:** плавная выпуклая кривая — минимум хорошо определён, оптимизация устойчива.
- **2D / 3D (contourf + surface):** овраг в районе найденного минимума, поверхность гладкая — признак хорошо обусловленной задачи.

                  opset_version=18,
                  input_names=['input'],
                  output_names=['output'],
                  dynamic_axes={'input': {0: 'batch_size'},
                                'output': {0: 'batch_size'}})
